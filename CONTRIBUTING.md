# Contributing to Soroban Airdrop Claim Template 🌊

First off, thank you for considering contributing to the Soroban Airdrop Claim Template! This project is proudly participating in the **Stellar Drips Wave**, which means you can earn rewards for your contributions.

## 🌊 How the Stellar Drips Wave Works
As part of the Drips Wave program, we maintain a backlog of issues that are eligible for rewards. 
1. **Find an Issue**: Look at our [Issues tab](https://github.com/OAKVISUALZ/soroban-airdrop-template/issues) for issues labeled `wave-eligible` or `good first issue`.
2. **Claim It**: Comment on the issue asking to be assigned so others know you are working on it.
3. **Submit a PR**: Fork the repo, make your changes, and submit a Pull Request (PR) linking to the issue.
4. **Earn Rewards**: Once your PR is merged during an active Wave cycle, you will earn points that translate to rewards from the Drips pool.

## 🚀 Getting Started

### 1. Fork & Clone
Fork the repository on GitHub and clone your fork locally:
```bash
git clone https://github.com/YOUR_USERNAME/soroban-airdrop-template.git
```

### 2. Branching Strategy
Create a new branch for your feature or bug fix:
```bash
git checkout -b feature/your-feature-name
```
or 
```bash
git checkout -b fix/your-bug-fix
```

### 3. Development Guidelines
- **Frontend (Next.js/React)**: Ensure your code follows the existing Tailwind CSS design patterns. Run `npm run lint` before committing.
- **Smart Contracts (Rust/Soroban)**: Write unit tests for any new contract logic you introduce using the Soroban SDK test utilities.

### 4. Creating a Pull Request
- Provide a clear and descriptive title for your PR.
- Reference the issue number you are solving (e.g., `Fixes #12`).
- Include screenshots if your PR introduces frontend UI changes.

We look forward to building the Stellar ecosystem with you!
